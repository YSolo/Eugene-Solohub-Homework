Lesson-01
=========

Git and GitHub
--------------

Set up your environment:

	git config --global user.name <NAME>
	git config --global user.email <EMAIL>
	
	# Display UTF-8 characters in filenames, if you're having problems seeing them
	git config --global core.quotepath off

	# auto-converting CRLF line endings into LF when you add a file to the index for Windows users
	git config --global core.autocrlf true 

	# or "true", git rejects irreversible conversions
	git config --global safecrlf true

	git config --list  // check what the values currently are

Use Git:

	git init
	git status
	git add <file name>
* one can use 'git add .' or 'git add -A' to commit all the files in the folder; -a for already posted files

	git reset <name>
	git checkout 
	git commit -m <text>  // m stands for 'message' to indicate a purpose of the commit 
* try to write commit message in imperative present tense

	git log // see log of commits

	git log --pretty=oneline
	git log --pretty=oneline --max-count=2
	git log --pretty=oneline --since='5 minutes ago'
	git log --pretty=oneline --until='5 minutes ago'
	git log --pretty=oneline --author=<your name>
	git log --pretty=oneline --all

	git log --all --pretty=format:"%h %cd %s (%an)" --since='7 days ago'
	
	git log --pretty=format:"%h %ad | %s%d [%an]" --graph --date=short
	Where:
		--pretty="..." defines the output format.
		%h is the abbreviated hash of the commit
		%d commit decorations (e.g. branch heads or tags)
		%ad is the commit date
		%s is the comment
		%an is the name of the author
		--graph tells git to display the commit tree in the form of an ASCII graph layout
		--date=short keeps the date format short and nice

	gitk // cool command to track the changes visualy.


	git diff //see the difference in unstaged and last commit
	git diff --staged // see the diff between staged and previous verion
	
Get help

	git help
	git help push
	

Clone project

	git clone git@github.com:UserName/RepoName.git
	git push origin master
